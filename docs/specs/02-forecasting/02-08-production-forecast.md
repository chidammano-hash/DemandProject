**Phase:** Evolution to Operations — Phase 1

**Status:** Not Started

## 1. Problem Statement

The ML models (LGBM, CatBoost, XGBoost) exist only in a backtesting context. Every backtest run trains models on historical data, evaluates them on held-out periods, and discards the trained weights. There is no mechanism to use those models to forecast months that have not yet happened.

Today, `fact_external_forecast_monthly` contains two types of rows:
- `model_id = 'external'` — forecasts from the source ERP system, covering historical and sometimes near-future periods
- `model_id = 'lgbm_cluster'` etc. — backtest predictions, which are predictions for historical periods where actuals already exist

**What is missing:** A row for Item 100320 at LOC 1401-BULK with `startdate = '2026-05-01'` (a future month) generated by the ML champion model.

### Concrete Failure Scenario

It is March 6, 2026. The inventory planner opens Demand Studio for Item 100320:
- On-hand qty: 120 units
- Safety stock: 60 units
- Last 3 months of sales: Jan=450, Feb=380, Mar=520
- The "Demand Forecast" panel is empty or shows only the external ERP forecast
- The external ERP forecast says Apr=400 (flat, no trend signal)
- The ML model, if it were run, would forecast Apr=490, May=510, Jun=470 (picking up a seasonal uptick)

The planner orders based on the ERP's 400 unit forecast. By April 18 the item stocks out. The ML signal existed but was never materialized.

## 2. Backtest vs. Production Forecast — Timeline Diagram

```
TIME AXIS ─────────────────────────────────────────────────────────────────────►

       2024-01    2024-06    2025-01    2025-06    2026-01    2026-04    2026-07
          │          │          │          │          │          │          │
          ▼          ▼          ▼          ▼          ▼          ▼          ▼
ACTUALS:  [============================KNOWN ACTUALS========================]
                                                             │
                                                     TODAY (2026-03-06)
BACKTEST  [TRAIN────────────────────────][PREDICT on known past][   ← NO DATA]
 RUNS:     each timeframe trains on left, predicts on right (all in the past)

EXTERNAL  [=========external ERP fcst rows=========][ERP near-future?][ empty ]
 FCST:

PRODUCTION                                                   [NEED THESE ROWS]
 FCST:                                                       [T+1  T+6   T+12]
```

**Key insight:** The backtest trains model weights and evaluates accuracy — but the weights are held in RAM only during the backtest run. The production inference pipeline must:
1. Persist model weights after backtest (currently discarded)
2. Build a future-oriented feature matrix (no actuals available — must use recursive inference)
3. Write predictions into `fact_production_forecast` with a versioned `plan_version`

## 4. Data Model

### 4.1 New Table: `fact_production_forecast`

This table stores forward-looking predictions only. Historical backtest rows remain in `fact_external_forecast_monthly`.

```sql
CREATE TABLE IF NOT EXISTS fact_production_forecast (
    id                  BIGSERIAL PRIMARY KEY,
    plan_version        VARCHAR(30)     NOT NULL,  -- e.g. '2026-03'
    item_no             VARCHAR(50)     NOT NULL,
    loc                 VARCHAR(50)     NOT NULL,
    forecast_month      DATE            NOT NULL,  -- first day of month, always future
    forecast_qty        NUMERIC(12, 2)  NOT NULL,
    forecast_qty_lower  NUMERIC(12, 2),            -- 10th percentile (if model supports)
    forecast_qty_upper  NUMERIC(12, 2),            -- 90th percentile
    model_id            VARCHAR(100)    NOT NULL,  -- e.g. 'lgbm_cluster', 'catboost_cluster'
    cluster_id          INTEGER,                   -- ml_cluster used for inference
    horizon_months      SMALLINT        NOT NULL,  -- 1=T+1, 2=T+2, ... 12=T+12
    is_recursive        BOOLEAN         NOT NULL DEFAULT FALSE,
    lag_source          VARCHAR(20),               -- 'actual' or 'predicted' (for lag_1)
    generated_at        TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    run_id              UUID            NOT NULL   -- ties rows to a single inference run
);

CREATE UNIQUE INDEX uq_prod_fcst_version_dfu_month
    ON fact_production_forecast (plan_version, item_no, loc, forecast_month);

CREATE INDEX idx_prod_fcst_item_loc_month
    ON fact_production_forecast (item_no, loc, forecast_month);

CREATE INDEX idx_prod_fcst_plan_version
    ON fact_production_forecast (plan_version);
```

**Grain:** one row per `(plan_version, item_no, loc, forecast_month)`
**Retention:** keep last 3 plan versions per DFU; older versions purged by cleanup job

### 4.2 New Table: `fact_model_registry`

Backtest scripts currently discard trained model weights. This table tracks where `.pkl` files are persisted so the inference pipeline can reload them.

```sql
CREATE TABLE IF NOT EXISTS fact_model_registry (
    id               BIGSERIAL PRIMARY KEY,
    model_id         VARCHAR(100)    NOT NULL,  -- e.g. 'lgbm_cluster'
    cluster_id       INTEGER         NOT NULL,
    timeframe        VARCHAR(20)     NOT NULL,  -- backtest timeframe that produced this model
    model_path       TEXT            NOT NULL,  -- e.g. 'data/models/lgbm_cluster/cluster_3.pkl'
    feature_cols     TEXT[],                    -- ordered list of feature column names used
    n_estimators     INTEGER,
    train_wape       NUMERIC(6, 4),
    trained_at       TIMESTAMPTZ     NOT NULL,
    is_active        BOOLEAN         NOT NULL DEFAULT FALSE,  -- TRUE = use for inference
    promoted_at      TIMESTAMPTZ,
    promoted_by      VARCHAR(100)
);

CREATE UNIQUE INDEX uq_model_registry_active
    ON fact_model_registry (model_id, cluster_id)
    WHERE is_active = TRUE;

CREATE INDEX idx_model_registry_model_id
    ON fact_model_registry (model_id, cluster_id, trained_at DESC);
```

```python
def persist_model(model, model_id: str, cluster_id: int, timeframe: str,
                  feature_cols: list, train_wape: float, config: dict) -> str:
    """
    Save trained model weights to disk and register in fact_model_registry.
    Returns the file path written.
    """
    import pickle, os
    base_path = config["model_registry"]["base_path"]
    out_dir = os.path.join(base_path, model_id)
    os.makedirs(out_dir, exist_ok=True)
    file_path = os.path.join(out_dir, f"cluster_{cluster_id}.pkl")
    with open(file_path, "wb") as f:
        pickle.dump({"model": model, "feature_cols": feature_cols}, f)
    # Register in DB (upsert: mark previous active=False, insert new)
    return file_path
```

### 5.2 New Script: `scripts/generate_production_forecasts.py`

```python
"""
generate_production_forecasts.py

Runs the champion model for each DFU over a forward horizon and writes
predictions to fact_production_forecast.

Usage:
    uv run python scripts/generate_production_forecasts.py [--horizon 12] [--dfu ITEM LOC] [--dry-run]

Functions:
    main()
    load_active_models(model_id: str, config: dict) -> dict[int, tuple[Any, list[str]]]
    get_champion_assignments(conn) -> pd.DataFrame
    build_inference_grid(item_no: str, loc: str, cluster_id: int,
                         sales_df: pd.DataFrame, horizon: int,
                         config: dict) -> pd.DataFrame
    generate_forecast_recursive(model, feature_cols: list, grid: pd.DataFrame,
                                 horizon: int) -> list[dict]
    write_forecast(rows: list[dict], plan_version: str, run_id: str,
                   conn, dry_run: bool) -> int
"""

import argparse, pickle, uuid, yaml, os
import pandas as pd
import numpy as np
from datetime import date, datetime
from dateutil.relativedelta import relativedelta
from common.db import get_db_params
import psycopg

CONFIG_PATH = "config/production_forecast_config.yaml"
ALGO_CONFIG_PATH = "config/algorithm_config.yaml"


def load_active_models(model_id: str, config: dict) -> dict:
    """
    Load pickled model artifacts for all clusters of a given model_id.
    Returns: {cluster_id: {"model": <model object>, "feature_cols": [...]}}
    """
    base_path = config["model_registry"]["base_path"]
    model_dir = os.path.join(base_path, model_id)
    models = {}
    for fname in os.listdir(model_dir):
        if not fname.endswith(".pkl"):
            continue
        cluster_id = int(fname.replace("cluster_", "").replace(".pkl", ""))
        with open(os.path.join(model_dir, fname), "rb") as f:
            artifact = pickle.load(f)
        models[cluster_id] = artifact
    return models


def get_champion_assignments(conn) -> pd.DataFrame:
    """
    Return the most recent champion model assignment per DFU.
    Joins dim_dfu to get cluster_assignment.
    """
    sql = """
        SELECT DISTINCT ON (f.dmdunit, f.loc)
            f.dmdunit  AS item_no,
            f.loc,
            f.model_id AS champion_model_id,
            d.cluster_assignment
        FROM fact_external_forecast_monthly f
        JOIN dim_dfu d ON d.dmdunit = f.dmdunit AND d.loc = f.loc
        WHERE f.model_id = 'champion'
        ORDER BY f.dmdunit, f.loc, f.startdate DESC
    """
    with conn.cursor() as cur:
        cur.execute(sql)
        rows = cur.fetchall()
    return pd.DataFrame(rows, columns=["item_no", "loc", "champion_model_id", "cluster_id"])

    For T+2: lag_1 = model's T+1 prediction (recursive write-back)
    For T+3: lag_1 = model's T+2 prediction, lag_2 = model's T+1 prediction

    The grid has one row per future month. Columns mirror the training feature set:
    qty_lag_1, qty_lag_2, qty_lag_3, qty_lag_4, qty_rolling_3, qty_rolling_6,
    month_sin, month_cos, ml_cluster, abc_class_enc, xyz_class_enc, seasonality_strength, etc.

    Returns a DataFrame with `horizon` rows, columns = feature_cols (no target column).
    """
    # ... (implementation calls common/feature_engineering.py helpers)
    pass


def generate_forecast_recursive(model, feature_cols: list,
                                  grid: pd.DataFrame, horizon: int) -> list:
    """
    Predict month-by-month, writing each prediction back as lag_1 for the next month.

    Returns list of dicts: [{forecast_month, forecast_qty, horizon_months, lag_source}, ...]
    """
    predictions = []
    for h in range(1, horizon + 1):
        row = grid.iloc[[h - 1]][feature_cols]
        pred = float(model.predict(row)[0])
        pred = max(0.0, pred)  # no negative forecasts
        lag_source = "actual" if h == 1 else "predicted"
        predictions.append({
            "forecast_month": ...,  # T + h months
            "forecast_qty": round(pred, 2),
            "horizon_months": h,
            "lag_source": lag_source,
        })
        # Write-back: update grid row h+1's lag_1 with this prediction
        if h < horizon:
            grid.at[h, "qty_lag_1"] = pred
            # recompute rolling features for next row...
    return predictions
```

- Consensus forecasting (merging ML forecast with commercial forecast — separate feature)
