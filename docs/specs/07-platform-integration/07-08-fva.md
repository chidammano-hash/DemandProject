# 07-08 Forecast Value Added (FVA) Tracking

## Overview

Tracks the value added at each step of the forecasting process -- from statistical baseline through planner overrides to final consensus. Measures whether each intervention improved or degraded accuracy, enabling data-driven process improvement.

## Components

| Component | Path | Purpose |
|---|---|---|
| Router | `api/routers/fva.py` | 3 REST endpoints for FVA analysis |
| Frontend | `frontend/src/tabs/FVATab.tsx` | FVA waterfall chart, intervention log, ROI summary |
| DDL | `sql/067_create_fva_tracking.sql` | `fact_fva_step`, `fact_forecast_intervention` |

## API Endpoints

| Method | Path | Description |
|---|---|---|
| GET | `/fva/waterfall` | FVA waterfall data: accuracy at each forecast step |
| GET | `/fva/interventions` | Planner intervention log with before/after accuracy |
| GET | `/fva/roi-summary` | Aggregate ROI of manual forecast adjustments |

## Database Schema

### `fact_fva_step`
- `step_id BIGSERIAL PRIMARY KEY`
- `item_no TEXT NOT NULL`, `loc TEXT NOT NULL`
- `month_start DATE NOT NULL`
- `step_name TEXT NOT NULL` (statistical, champion, bias_corrected, planner_override, consensus)
- `step_order INTEGER NOT NULL` (1-5, defines waterfall sequence)
- `forecast_qty NUMERIC NOT NULL`
- `actual_qty NUMERIC` (populated after actuals arrive)
- `wape NUMERIC`, `bias_pct NUMERIC`
- `computed_at TIMESTAMPTZ DEFAULT NOW()`
- Unique: `(item_no, loc, month_start, step_name)`
- Indexes: `(item_no, loc, month_start)`, `(step_name)`

### `fact_forecast_intervention`
- `intervention_id BIGSERIAL PRIMARY KEY`
- `item_no TEXT NOT NULL`, `loc TEXT NOT NULL`
- `month_start DATE NOT NULL`
- `intervener TEXT NOT NULL` (user or process name)
- `from_step TEXT NOT NULL`, `to_step TEXT NOT NULL`
- `original_qty NUMERIC NOT NULL`, `adjusted_qty NUMERIC NOT NULL`
- `adjustment_pct NUMERIC` (computed: `(adjusted - original) / original * 100`)
- `reason TEXT`
- `accuracy_before NUMERIC`, `accuracy_after NUMERIC` (populated retroactively)
- `value_added NUMERIC` (accuracy_after - accuracy_before; positive = improvement)
- `created_at TIMESTAMPTZ DEFAULT NOW()`
- Indexes: `(item_no, loc, month_start)`, `(intervener)`, `(value_added)`

## FVA Waterfall

The waterfall shows cumulative accuracy at each forecasting step:

1. **Statistical** — raw model output (e.g., LGBM/CatBoost champion)
2. **Champion** — after champion model selection
3. **Bias-corrected** — after bias correction engine (F3.1)
4. **Planner override** — after manual planner adjustments
5. **Consensus** — final agreed forecast (S&OP output)

Each step's WAPE and bias are computed against actuals. The waterfall visualizes where accuracy improves or degrades.

## ROI Summary

Aggregates intervention value across all DFUs:

- Total interventions count
- Interventions that improved accuracy vs. degraded
- Net accuracy gain/loss from manual touches
- Average adjustment magnitude
- Top interveners ranked by net value added

## Frontend (`FVATab.tsx`)

- **Waterfall chart**: Recharts stacked bar chart showing accuracy gain/loss per step
- **Intervention log table**: Sortable/filterable table of all planner adjustments with value-added indicator
- **ROI KPI cards**: Net Value Added %, Improvement Rate, Avg Adjustment Size
- **Filters**: Item/location selector, date range, intervener filter
- Sidebar nav item: "FVA" with `GitCompare` icon

## Integration Points

- `scripts/generate_production_forecasts.py` writes the `statistical` and `champion` FVA steps
- Bias correction engine (F3.1) writes the `bias_corrected` step
- Override queue panel writes `planner_override` step on save
- S&OP tab writes `consensus` step on approval
- Actuals backfill job retroactively computes accuracy and value_added
