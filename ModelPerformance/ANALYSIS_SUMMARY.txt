│     └─ 5–15 min per cluster fit (CatBoost.fit())

WHY SO CONSERVATIVE:
├─ Lossguide growth policy → 3000 iterations (vs. 1500 LGBM, 2000 XGBoost)

  └─ Rationale: Lossguide still has 2500 iters (sufficient), early stopping better
  └─ Impact: -17% iterations, -20% wall time per cluster
  └─ Risk: 🟢 Low (still conservative)

Change 2: learning_rate: 0.008 → 0.010
  └─ Rationale: Bump to middle ground; 0.008 was very conservative
  └─ Impact: 15% faster convergence per cluster
  └─ Risk: 🟡 Medium (validate on single TF)

Change 3: border_count: 64 → 32
  └─ Rationale: 32 bins sufficient for 62 features on monthly grain
  └─ Impact: 25% faster histogram computation
  └─ Risk: 🟢 Low (<1% accuracy loss expected)

Change 4: max_leaves: 127 → 100
  └─ Rationale: Fewer leaves per tree, faster iteration
  └─ Impact: 10% smaller tree size
  └─ Risk: 🟢 Low

Change 5: leaf_estimation_iterations: 10 → 8
  └─ Rationale: Newton leaf refinement already good at 8 iterations
  └─ Impact: 20% fewer leaf refinement loops
  └─ Risk: 🟢 Low (<0.5% accuracy loss)

Change 6: model_size_reg: 0.08 → 0.10
  └─ Rationale: Higher L1 penalty on tree size = faster convergence
  └─ Impact: Discourages large trees
  └─ Risk: 🟢 Low

COMBINED EFFECT:
├─ Wall-time reduction: 25–35%
├─ Accuracy loss: -0.5% to +0.5% (likely neutral)
├─ Memory reduction: 10% (fewer leaves)
└─ Implementation time: 5 minutes

PARALLELIZATION ASSESSMENT
================================================================================

Current Parallelization: ✅ Enabled (ProcessPoolExecutor, 8 workers)
├─ Per-cluster within each timeframe
├─ 6–7× speedup on 8-core system
├─ Safe (no shared state, picklable data)
└─ Status: GOOD, no changes needed

Timeframe Parallelization: ❌ Not implemented (sequential loop)
├─ Would require 10× data grid copies (~10 GB memory overhead)
├─ Potential: 2.5–3× speedup if memory available
├─ Risk: Data consistency, pickle overhead (~100ms per TF)
└─ Recommendation: Not worth now (per-cluster already parallelized)

GPU Acceleration: ❌ Disabled (correct for small clusters)
├─ Per-cluster rows: 10–200 (too small for GPU benefit)
├─ GPU overhead: ~100ms per cluster (overhead > benefit)
├─ Recommendation: KEEP DISABLED for monthly grain
└─ Only enable if cluster rows > 50K (not current case)
